## Task 1: Migration â€” 2 SECURITY DEFINER RPCs + index

**Codex plan-pass CRITICAL â€” schema-qualify everything.** RPCs reference `audit.events`, `public.runs`, `public.memberships`, `auth.users`. Index also references `audit.events`. Each function declares `SET search_path = public, audit, auth, pg_temp` so SECURITY DEFINER bypasses do not pivot through user-controlled schemas. Test 27 asserts `SET search_path` line + `audit.events` schema qualification.

**Codex plan-pass CRITICAL â€” runs.cost_usd dependency.** This delta MUST run after Phase 4's `20260508120000_phase4_runs_metadata.sql`. Filename timestamp ordering enforces this; Test 27 also greps for the Phase 4 file's existence in `data/deltas/` to fail fast if missing.

See spec for full SQL. Key points:
- Index `audit_events_org_keyset_idx` on `audit.events (organization_id, occurred_at DESC, id DESC)`
- `list_audit_events(p_caller_user_id, p_org_id, p_cursor_occurred_at, p_cursor_id, p_limit, p_action, p_actor_user_id, p_since, p_until) RETURNS jsonb`
  - Authorize first via memberships lookup; raise `not_admin` (P0001) if NULL or not in admin/owner
  - Clamp limit to [1, 200]
  - Keyset paginate: `WHERE (e.occurred_at, e.id) < (p_cursor_occurred_at, p_cursor_id)` when cursor present
  - LEFT JOIN auth.users for actor email
  - Return `{ events: [...], nextCursor: { occurredAt, id } | null }`
- `org_cost_report(p_caller_user_id, p_org_id, p_since, p_until, p_group_by) RETURNS jsonb`
  - p_group_by must equal 'user'; raise `bad_group_by` else
  - Authorize via memberships; raise `not_admin` if not admin/owner
  - Aggregate runs WHERE organization_id AND deleted_at IS NULL AND created_at >= p_since AND created_at < p_until GROUP BY user_id
  - Use `coalesce(sum(coalesce(col, 0)), 0)` to NULL-safe both directions
  - LEFT JOIN auth.users for email
  - Return `{ rows: [...], total: {...}, period: { since, until } }`
- REVOKE ALL ... FROM PUBLIC, anon, authenticated; GRANT EXECUTE ... TO service_role
