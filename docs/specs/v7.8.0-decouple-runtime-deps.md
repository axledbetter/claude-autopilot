# v7.8.0 — Decouple `tsx` + Supabase from default install; trim snapshots

## Why now

Codex weight-review (2026-05-11) confirmed:

1. **`tsx`** is the single biggest installed-size contributor (pulls
   `esbuild` ~10MB + platform binaries). Only a subset of users
   actually runs `.ts` scripts.
2. **`@supabase/supabase-js`** is only needed for hosted-dashboard
   upload (`--upload-to-dashboard`). Local-only users — the
   majority — pay for it on every install.
3. **`tests/snapshots/`**, **`scripts/snapshots/`**, and
   **`scripts/autoregress.ts`** ship in the npm tarball but end
   users never read them.

This release decouples both runtime deps without breaking existing
users, and trims tarball noise. **No public API or CLI behavior
changes for the default path.** A deprecation warning is the only
new user-visible behavior.

## Non-goals

* Removing `tsx` from `dependencies` — v8.0.0.
* Splitting `@delegance/claude-autopilot-cloud` — v8.0.0.
* Bundling `dist/src` with esbuild — out of scope; measure
  cold-start first.
* Moving LLM SDKs to dynamic-import — v7.9.0.

## Scope

### Part 1: tsx resolution + deprecation warning

**Cross-platform tsx resolver.** Returns a `{command, args, source}`
triple so spawning is portable (Windows uses `.cmd` shims for bin
entries, Unix uses direct shebangs — both work when the bin name
goes through `command`, not by spawning an `.mjs` directly).

```ts
// src/cli/tsx-resolver.ts (new, ~140 LOC)
import { createRequire } from 'node:module';
import { existsSync, readFileSync } from 'node:fs';
import path from 'node:path';

export interface TsxResolution {
  /** Command to spawn (absolute path or PATH-resolvable name) */
  command: string;
  /** Args prepended before the user script path */
  args: string[];
  /** Where tsx came from — used for deprecation warning */
  source: 'project-local' | 'path' | 'bundled';
  /** Override origin if env var was set */
  forcedBy?: 'env' | 'flag';
}

export interface ResolveOpts {
  projectRoot: string;
  /** Read CLAUDE_AUTOPILOT_TSX env var */
  envOverride?: string;
  /** --tsx-source=bundled|project|path CLI flag */
  flagOverride?: 'bundled' | 'project' | 'path';
  /** Suppress filesystem dedup writes (test helper) */
  suppressWarning?: boolean;
}

export function resolveTsx(opts: ResolveOpts): TsxResolution {
  const { projectRoot, envOverride, flagOverride } = opts;

  // --- Escape hatch: explicit override ---
  if (flagOverride) {
    return resolveFromSource(flagOverride, projectRoot, 'flag');
  }
  if (envOverride && ['bundled', 'project', 'path'].includes(envOverride)) {
    return resolveFromSource(envOverride as any, projectRoot, 'env');
  }

  // --- Normal precedence ladder ---
  // 1. Project-local
  const projectResolved = tryProjectLocal(projectRoot);
  if (projectResolved) return projectResolved;

  // 2. PATH (cross-platform via npm `which` package — added to deps)
  const pathResolved = tryPath();
  if (pathResolved) return pathResolved;

  // 3. Bundled (deprecation warning)
  if (!opts.suppressWarning) {
    emitTsxDeprecationWarningSafe();
  }
  return resolveBundled();
}

function tryProjectLocal(projectRoot: string): TsxResolution | null {
  try {
    const projectRequire = createRequire(
      path.join(projectRoot, 'package.json'),
    );
    const pkgPath = projectRequire.resolve('tsx/package.json');
    return buildResolutionFromPkgJson(pkgPath, 'project-local');
  } catch {
    return null;
  }
}

function tryPath(): TsxResolution | null {
  // Use `which` npm package — cross-platform, no shell-out
  try {
    const which = require('which') as typeof import('which');
    const tsxBin = which.sync('tsx', { nothrow: true });
    if (tsxBin) {
      return { command: tsxBin, args: [], source: 'path' };
    }
  } catch {
    // `which` not installed — skip
  }
  return null;
}

function resolveBundled(): TsxResolution {
  const bundledRequire = createRequire(import.meta.url);
  const pkgPath = bundledRequire.resolve('tsx/package.json');
  return buildResolutionFromPkgJson(pkgPath, 'bundled');
}

/**
 * Read tsx's package.json and resolve the declared bin entry.
 * Falls back to spawning via `node <cliJs>` so it works on Windows
 * without relying on shebang execution.
 */
function buildResolutionFromPkgJson(
  pkgPath: string,
  source: TsxResolution['source'],
): TsxResolution {
  const pkgDir = path.dirname(pkgPath);
  const pkg = JSON.parse(readFileSync(pkgPath, 'utf8'));

  // pkg.bin can be string OR object — both are valid in npm spec
  let binRel: string | undefined;
  if (typeof pkg.bin === 'string') binRel = pkg.bin;
  else if (pkg.bin && typeof pkg.bin === 'object') binRel = pkg.bin.tsx;

  if (!binRel) {
    throw new Error(`tsx package.json at ${pkgPath} has no bin.tsx entry`);
  }

  const binAbs = path.resolve(pkgDir, binRel);
  if (!existsSync(binAbs)) {
    throw new Error(`tsx bin ${binAbs} declared in package.json but missing on disk`);
  }

  // Spawn via node directly — portable across Windows/Unix, avoids
  // shebang/`.cmd` shim quirks, no PATHEXT issues.
  return { command: process.execPath, args: [binAbs], source };
}

function resolveFromSource(
  source: 'bundled' | 'project' | 'path',
  projectRoot: string,
  forcedBy: 'env' | 'flag',
): TsxResolution {
  let result: TsxResolution | null = null;
  if (source === 'project') result = tryProjectLocal(projectRoot);
  else if (source === 'path') result = tryPath();
  else result = resolveBundled();

  if (!result) {
    throw new Error(
      `tsx source=${source} requested via ${forcedBy} but resolution failed. ` +
        `Install tsx in your project or unset the override.`,
    );
  }
  return { ...result, forcedBy };
}
```

**Deprecation warning** — once per calendar day, dedup at
`~/.claude-autopilot/.tsx-deprecation-shown`. **Non-fatal on
filesystem errors** (CI, read-only home, containers):

```ts
function emitTsxDeprecationWarningSafe(): void {
  // Allow CI/log-hygiene opt-out
  if (process.env.CLAUDE_AUTOPILOT_NO_TSX_DEPRECATION === '1') return;

  try {
    const dedupPath = path.join(
      os.homedir(),
      '.claude-autopilot',
      '.tsx-deprecation-shown',
    );
    const today = new Date().toISOString().slice(0, 10); // YYYY-MM-DD
    if (existsSync(dedupPath)) {
      const lastShown = readFileSync(dedupPath, 'utf8').trim();
      if (lastShown === today) return;
    }
    mkdirSync(path.dirname(dedupPath), { recursive: true });
    writeFileSync(dedupPath, today);
  } catch {
    // Non-fatal — print the warning anyway, don't error the run.
  }

  process.stderr.write(TSX_DEPRECATION_MESSAGE + '\n');
}

const TSX_DEPRECATION_MESSAGE = `
⚠️  @delegance/claude-autopilot is using its bundled \`tsx\` to run
    your TypeScript scripts. In v8.0.0, \`tsx\` will be removed from
    runtime deps and you'll need to install it yourself:

        npm install -D tsx

    To silence this warning now and prepare for v8.0.0:
      1. Add \`tsx\` to your project's devDependencies, OR
      2. Set \`CLAUDE_AUTOPILOT_NO_TSX_DEPRECATION=1\` in your env.

    Override resolution: \`CLAUDE_AUTOPILOT_TSX=bundled|project|path\`
`;
```

### Part 2: Supabase lazy-load

**Rule**: no *value* imports of `@supabase/supabase-js` in modules
reachable from CLI bootstrap. `import type { ... } from
'@supabase/supabase-js'` is fine (TypeScript erases type imports).
All runtime usage must go through `await import(...)`.

Audit step (run as part of CI):
```bash
# Grep for any non-type imports of supabase in src/, dist/.
# Allow only `import type`.
npx tsx scripts/audit-supabase-imports.ts
# Exits 1 if any value-level static import found outside permitted
# lazy-load modules.
```

Conversion sites — every `import { createClient } from
'@supabase/supabase-js'` becomes:

```ts
// Before:
import { createClient } from '@supabase/supabase-js';

export async function uploadRun(args: UploadArgs) {
  const supa = createClient(args.url, args.key);
  // ...
}

// After:
import type { SupabaseClient } from '@supabase/supabase-js';

export async function uploadRun(args: UploadArgs) {
  let supa: SupabaseClient;
  try {
    const mod = await import('@supabase/supabase-js');
    supa = mod.createClient(args.url, args.key);
  } catch (err) {
    if (isMissingOptionalPackageError(err, '@supabase/supabase-js')) {
      throw new Error(
        `Dashboard upload requires @supabase/supabase-js. Install with:\n\n` +
          `    npm install @supabase/supabase-js\n`,
      );
    }
    throw err;
  }
  // ...
}
```

**Missing-package detection** that doesn't misclassify transitive
failures:

```ts
// src/cli/dashboard/missing-package.ts (new, ~30 LOC)
export function isMissingOptionalPackageError(
  err: unknown,
  pkgName: string,
): boolean {
  if (!(err instanceof Error)) return false;
  const e = err as NodeJS.ErrnoException & { url?: string };
  // Node ESM dynamic import failure
  if (e.code !== 'ERR_MODULE_NOT_FOUND' && e.code !== 'MODULE_NOT_FOUND') {
    return false;
  }
  // Inspect the specifier — if it isn't our package, don't claim it is.
  // err.message format: "Cannot find package '<pkg>' imported from <url>"
  // or for CJS: "Cannot find module '<pkg>'"
  const specifier = extractMissingSpecifier(e.message);
  return specifier === pkgName;
}

function extractMissingSpecifier(msg: string): string | undefined {
  const m =
    msg.match(/Cannot find package '([^']+)'/) ||
    msg.match(/Cannot find module '([^']+)'/);
  return m?.[1];
}
```

Then move `@supabase/supabase-js` from `dependencies` to
`optionalDependencies`. **Does not reduce default install size**
(npm installs optional deps by default), but enables
`npm install --omit=optional` for local-only users and tees up
v8.0.0 extraction into `@delegance/claude-autopilot-cloud`.

## Risk + rollback

**Risk vectors:**

1. **tsx resolution order regression** — project-local resolves to
   broken/incompatible `tsx`. Mitigation: escape hatch
   (`CLAUDE_AUTOPILOT_TSX=bundled`), clear error message naming
   which source resolved.
2. **Supabase dynamic-import in restricted environments** — Node
   22+ ESM supports dynamic import universally; not a concern for
   our supported runtime.
3. **`which` dep adds 18 KB** — tiny, but it's a new direct dep.
   Alternative: hand-roll cross-platform PATH lookup (~30 LOC).
   Recommend using `which` (battle-tested, npm-internal use it).
4. **Snapshot files referenced at runtime** — audit step above
   covers this. Add CI grep to lock it in.

**Rollback**: revert the PR. No data migrations, no protocol
changes.

## Amendments from codex pass 2 (2026-05-11)

### A1 [CRITICAL]: ESM/CJS module-system safety in `tsx-resolver.ts`

The resolver source uses `import.meta.url` (ESM) and `require()`
(CJS). Mixing these fails depending on emit target. Fix: define
`require` via `createRequire(import.meta.url)` at module scope.
Compiled output runs as ESM (`"type": "module"` in package.json).

```ts
import { createRequire } from 'node:module';
const require = createRequire(import.meta.url);
// ...then `require('which')` is safe in the ESM-compiled file.
```

Add a unit test that imports the compiled `dist/cli/tsx-resolver.js`
and runs `resolveTsx()` against a fixture project — fails if module
loading throws.

### A2 [CRITICAL]: CLI parser scope for `--tsx-source`

Implementation must explicitly:
1. Register `--tsx-source <bundled|project|path>` in the CLI option
   schema next to other top-level flags
2. Validate the value, error actionably on bad input (e.g.,
   `--tsx-source=foo` → `Invalid --tsx-source value 'foo'. Expected
   bundled, project, or path.`)
3. Include in `--help` output under a "Resolution overrides" group
4. Add one CLI-level test: `claude-autopilot --tsx-source=bundled
   run <script>` succeeds and forwards to resolver

### A3 [WARNING]: PATH-resolved tsx may be the bundled copy

When `tsx` is invoked via `npm exec` / globally-installed shims,
`which.sync('tsx')` can resolve to a path inside *our own*
`node_modules/.bin`. That would incorrectly mark resolution as
`source: 'path'` and suppress the deprecation warning.

Fix: after PATH resolution, normalize the result against
`resolveBundled().command`/`args[0]` directory. If the PATH-resolved
bin lives within our package's `node_modules`, treat as
`source: 'bundled'` (and warn).

```ts
function tryPath(): TsxResolution | null {
  const tsxBin = which.sync('tsx', { nothrow: true });
  if (!tsxBin) return null;

  // Is this PATH entry actually pointing at our own bundled tsx?
  const bundled = resolveBundled();
  const bundledDir = path.dirname(bundled.args[0]);
  if (path.resolve(tsxBin).startsWith(path.resolve(bundledDir))) {
    return null; // fall through to bundled (with warning)
  }
  return { command: tsxBin, args: [], source: 'path' };
}
```

Add test: PATH points at bundled `node_modules/.bin/tsx` → treated
as bundled, warning emitted.

### A6 [WARNING]: Hand-roll PATH lookup, drop `which` dep

Reconsidering: adding `which` (a new direct dep with its own
maintenance surface) for a 10-line cross-platform PATH lookup is
overkill. Hand-roll instead:

```ts
function tryPath(): TsxResolution | null {
  const PATH = process.env.PATH ?? '';
  const PATHEXT = process.platform === 'win32'
    ? (process.env.PATHEXT ?? '.EXE;.CMD;.BAT').split(';')
    : [''];
  const sep = process.platform === 'win32' ? ';' : ':';

  for (const dir of PATH.split(sep)) {
    for (const ext of PATHEXT) {
      const candidate = path.join(dir, `tsx${ext}`);
      if (existsSync(candidate)) {
        // Apply A3 self-pointer check here
        // ...
        return { command: candidate, args: [], source: 'path' };
      }
    }
  }
  return null;
}
```

Drops the `which` dep entirely. Tests cover Unix + Windows code
paths via `process.platform` mocking.
