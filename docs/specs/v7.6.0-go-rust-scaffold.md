# v7.6.0 — Go + Rust scaffold support

## Goal

Closes the v7.4.0 deferred items (Go/Rust). Same shape as
v7.4.0 Python/FastAPI: each stack gets its own scaffolder module
under `src/cli/scaffold/`, the dispatcher detects + routes, the
`--stack` flag overrides.

Risk: **low** (additive scaffolder, no auth/data-access touched).
Per v7.4.2 policy → 1 codex pass.

## Stack detection (extends v7.4.0)

Detection precedence ladder, extended:

1. `--stack <value>` explicit flag overrides (validated set:
   `node | python | fastapi | go | rust`; unknown → exit 3).
2. **FastAPI** (path heuristic + prose mention, existing).
3. **Python** (path heuristic, existing).
4. **Go** (NEW): `## Files` lists `go.mod` OR `main.go` OR
   `cmd/<n>/main.go`.
5. **Rust** (NEW): `## Files` lists `Cargo.toml` OR
   `src/main.rs` OR `src/lib.rs`.
6. **Node ESM** (existing): `package.json`.
7. Detected-but-unsupported now only `Gemfile` (v7.7+).
8. Fallback: Node ESM.

## Go scaffolding

When stack = `go`:

* **`go.mod`** — module name = PEP-503-style normalized
  `basename(cwd)` BUT keep dots/hyphens (Go module names are
  permissive; just lowercase). Go directive = `1.22`.
* **`main.go` (cmd/<name>/main.go if spec lists it, else root)**:
  ```go
  package main

  import "fmt"

  func main() {
      fmt.Println("Hello from <module_name>")
  }
  ```
* **`main_test.go`** alongside main.go:
  ```go
  package main

  import "testing"

  func TestMain(t *testing.T) {
      // Replace with real cases.
      _ = t
  }
  ```
* Default test command: `go test ./...`.
* `.gitignore` augmentations: append `vendor/`, `*.exe`,
  `*.test` (idempotent — only add if missing).

## Rust scaffolding

When stack = `rust`:

* **`Cargo.toml`** — PEP-503-ish normalized package name
  (lowercase, hyphens replaced with underscores for Rust crate
  identifier rules); version `0.1.0`, edition `2021`.
* **`src/main.rs`** (binary crate):
  ```rust
  fn main() {
      println!("Hello from {}", env!("CARGO_PKG_NAME"));
  }
  ```
* **`tests/integration_test.rs`**:
  ```rust
  #[test]
  fn smoke() {
      assert_eq!(2 + 2, 4);
  }
  ```
* Default test command: `cargo test`.
* `.gitignore` augmentation: append `target/`, `Cargo.lock`
  conditionally (the convention is keep Cargo.lock for binaries,
  ignore for libraries — heuristic: if spec lists `src/lib.rs`
  alongside `src/main.rs`, add `Cargo.lock` to .gitignore; if
  binary-only, don't).

## Dispatcher changes

`src/cli/scaffold.ts` (dispatcher) gets two new cases:

```ts
case 'go':
  return scaffoldGo({ cwd, parsed, dryRun });
case 'rust':
  return scaffoldRust({ cwd, parsed, dryRun });
```

`--list-stacks` output extended:

```
Supported: node, python, fastapi, go, rust
Auto-detected from `## Files`:
  node    (package.json)
  python  (pyproject.toml / requirements.txt)
  fastapi (main.py + fastapi mention)
  go      (go.mod / main.go / cmd/*/main.go)
  rust    (Cargo.toml / src/main.rs / src/lib.rs)
Recognized-but-unsupported (exit 3): Gemfile (v7.7+)
```

## Tests

1. Detection from `go.mod` only
2. Detection from `cmd/foo/main.go` (cmd-pattern)
3. End-to-end: empty repo → `go build` succeeds (codex N3
   integration; skipped if `go` not on PATH)
4. Module name normalization (`my-pkg-2` → `my-pkg-2` lowercase
   preserved — Go allows hyphens)
5. `.gitignore` augmentation idempotent
6. Existing `go.mod` preserved (never overwrite)
7. `--stack go` override on a non-Go spec
8. Spec listing both `go.mod` AND `package.json` → exit 3 polyglot
9. `cmd/<name>/main.go` directory created
10. Test runner config: `go test ./...` in CLAUDE.md if generated

10 Rust tests (analogous):
1. Detection from `Cargo.toml`
2. Detection from `src/main.rs`
3. Lib vs bin detection — `src/lib.rs` present → adds
   `Cargo.lock` to .gitignore
4. End-to-end: empty repo → `cargo check` succeeds (skipped
   if `cargo` not on PATH)
5. Crate name normalization (`my-pkg-2` → `my_pkg_2` for crate
   identifier rules)
6. `Cargo.toml` package name + crate name consistent
7. Existing `Cargo.toml` preserved
8. `--stack rust` override
9. Polyglot exit 3
10. integration test file exists at `tests/integration_test.rs`

4 stack-detection precedence regression cases (Go beats Python
when both present? — no, that's polyglot → 3. Single-stack only).

## Out of scope

* Workspace specs (`workspace.toml`, multi-crate). Single-crate
  Rust + single-module Go only.
* Custom `go.work` files.
* Rust `dev-dependencies` parsing from spec prose (the v7.4.0
  narrow contract applies: 3 patterns only).
