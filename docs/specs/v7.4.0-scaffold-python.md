# v7.4.0 spec — `scaffold --from-spec` per-stack support

## Goal

Extend `claude-autopilot scaffold --from-spec <path>` (shipped v7.2.0,
Node-ESM only) to detect target stack from the spec markdown and emit
the appropriate skeleton. v7.4.0 adds **Python** + **Python/FastAPI**.

Closes the v7.1.6 + v7.1.8 benchmark caveat ("n=1, Node 22 ESM only —
Python/Rust/Go remain v8 follow-ups"). Per-stack scaffolders are also
listed in the v8 spec stabilization criteria #2 (4-repo benchmark suite
needs more than just blank-Node).

## Stack detection

Detection happens AFTER parsing the spec's `## Files` section.
**Precedence (codex CRITICAL #1):** explicit `--stack` flag → FastAPI →
Python → Node → unsupported-detected → Node fallback. Each rung is
checked in order; first match wins.

1. **`--stack <value>`** explicit flag overrides everything below
   (validated against `node|python|fastapi`; unknown → exit 3).
2. **FastAPI** (checked BEFORE generic Python so FastAPI specs that
   also list `pyproject.toml` aren't mis-classified): `## Files` lists
   `app/main.py` OR `main.py` OR `src/<pkg>/main.py` AND any bullet
   description contains `fastapi` (case-insensitive).
3. **Python**: `## Files` lists `pyproject.toml` OR `requirements.txt`.
4. **Node ESM** (existing v7.2.0): `## Files` lists `package.json`.
5. **Detected-but-unsupported (codex W2)**: `## Files` lists `go.mod`
   OR `Cargo.toml` OR `Gemfile` → exit 3 with diagnostic
   `"<stack> detected but not supported until v7.5"`. Does NOT
   silently fallback to Node — that would generate broken scaffolding
   the operator didn't ask for.
6. **Fallback**: Node ESM (preserves v7.2.0 default for ambiguous
   path-only specs).

**Ambiguity (codex WARNING #3):** if both `package.json` AND
`pyproject.toml` are listed in `## Files`, exit 3 with
`"polyglot spec — pass --stack to disambiguate"`. v7.4.0 only
supports single-stack scaffolding; polyglot is v7.5+.

## Python scaffolding

When stack is detected as Python:

* **`pyproject.toml`** — PEP 621 metadata, sensible defaults:
  ```toml
  [project]
  name = "<distribution_name>"     # PEP 503 normalized
  version = "0.1.0"
  requires-python = ">=3.11"
  dependencies = [...]              # parsed from spec hints (see below)

  [build-system]
  requires = ["hatchling"]
  build-backend = "hatchling.build"

  [tool.hatch.build.targets.wheel]
  packages = ["src/<package_name>"]  # codex W1: explicit, no auto-discovery

  [tool.pytest.ini_options]
  testpaths = ["tests"]
  ```

* **Name normalization (codex WARNING #1):**
  - `distribution_name` = PEP 503 normalized form of `basename(cwd)`:
    lowercase, hyphens preserved, runs of `[._-]+` → `-`, strip
    leading/trailing `[._-]`.
  - `package_name` = valid Python identifier:
    `distribution_name` → replace `-`/`.` with `_`, prefix `_` if
    starts with a digit. `my-pkg-2` → `my_pkg_2`; `2cool` → `_2cool`.
  - Both deterministic; tests pin the algorithm.

* **`README.md`** (codex NOTE #1): always create. If `## Files` lists
  it, write a placeholder mentioning the spec source; otherwise
  generic placeholder. Never overwrite an existing README.

* **`src/<package_name>/__init__.py`** — empty file (so the package
  is importable immediately).

* **Directories**: `src/<package_name>/`, `tests/`.

* **Polyglot guard (codex WARNING #5):** if `## Files` ALSO lists
  `package.json` / `tsconfig.json` and `--stack` was NOT passed,
  exit 3 with the polyglot diagnostic above (already covered in
  detection step 6). When `--stack python` IS passed and Node
  files are present in the spec, scaffolder warns
  `"Ignoring Node files (--stack python): package.json, tsconfig.json"`
  and skips them.

## FastAPI scaffolding

When stack is FastAPI:

* All Python defaults above, plus:

* **`pyproject.toml`** auto-includes `fastapi>=0.110` and
  `uvicorn[standard]>=0.27` in `dependencies` (deduped against any
  spec-supplied hints — if the spec already lists `fastapi` at any
  version, we don't override; we only add when missing).

* Adds `[project.scripts] <distribution_name>-server = "<package_name>.main:run"`.

* **`src/<package_name>/main.py`** (codex CRITICAL #2 — this was
  missing, leaving the entrypoint dangling). Generated content:
  ```python
  """FastAPI entrypoint — auto-scaffolded by claude-autopilot.
  Override the prose docstring + add real routes; keep `app` and
  `run()` exported so the [project.scripts] entry stays valid.
  """
  from fastapi import FastAPI
  import uvicorn

  app = FastAPI()


  @app.get("/health")
  def health() -> dict[str, str]:
      return {"status": "ok"}


  def run() -> None:
      uvicorn.run("<package_name>.main:app", host="0.0.0.0", port=8000)
  ```
  Operator must add real routes; the `/health` stub is just so the
  scaffold is genuinely runnable + the entrypoint actually resolves.

* **`tests/test_main.py`** (auto-included even if spec doesn't list
  it explicitly — without it, the whole `pyproject.toml`
  `[tool.pytest.ini_options]` block is dead config). Generated:
  ```python
  from fastapi.testclient import TestClient
  from <package_name>.main import app

  client = TestClient(app)


  def test_health() -> None:
      response = client.get("/health")
      assert response.status_code == 200
      assert response.json() == {"status": "ok"}
  ```

* Default test command is `pytest`.

## Dependency hint extraction (codex WARNING #6 — narrowed contract)

For Python/FastAPI specs, dependency extraction uses ONLY this
narrow set of patterns (everything else ignored):

1. **Explicit `dependencies: [...]` block** in spec prose
   (existing v7.2.0 parser, but values flow into Python deps array
   instead of npm `dependencies`). Each entry preserved verbatim.
2. **Backticked package names with extras**:
   `` `uvicorn[standard]` `` → adds as-is (no version inferred).
3. **Phrase `depends on <name>`** in any bullet — adds `<name>` as
   bare requirement.

Never infer version specifiers from prose. Never apply known-package
heuristics. Dedupe by PEP 503 normalized name; first occurrence wins
the version specifier (if any).

## Spec parser additions

`parseSpecFiles` already extracts paths from backticks. New: extract
**stack hints** from prose (e.g. "uses fastapi", "Python 3.12",
"Node 22 ESM"). Returns optional `stackHint` field on
`ParsedFiles.packageHints`. Used as a tie-breaker if path heuristics
are ambiguous.

## CLI changes

* `scaffold --from-spec <path>` (existing): unchanged behavior on
  Node specs; auto-detects Python/FastAPI now.
* `scaffold --from-spec <path> --stack python` (new): force Python
  even if detection would pick Node.
* `scaffold --from-spec <path> --stack fastapi` (new): force FastAPI.
* `--list-stacks` (new, codex NOTE #2): print three sections:
  - **Supported** (`--stack` accepts these): `node`, `python`, `fastapi`
  - **Auto-detected from `## Files`**: `node` (`package.json`),
    `python` (`pyproject.toml` / `requirements.txt`), `fastapi`
    (`main.py` + fastapi mention)
  - **Recognized-but-unsupported (exit 3)**: `go.mod` (v7.5),
    `Cargo.toml` (v7.5), `Gemfile` (v7.5+)

Exit codes:
* 0 — scaffolded
* 1 — spec file missing
* 2 — spec missing `## Files` section
* 3 (NEW) — `--stack` value not recognized,
  detected-but-unsupported stack (Go/Rust/Ruby), or
  polyglot spec without explicit `--stack`

## Out of scope (v7.5+)

* Go (`go mod init`, `main_test.go`).
* Rust (`cargo new`).
* Ruby/Rails.
* JavaScript-without-package.json (single-file CLI scripts).
* TypeScript with build step (already handled by `--stack node`
  detecting `.ts` paths in v7.2.0).
