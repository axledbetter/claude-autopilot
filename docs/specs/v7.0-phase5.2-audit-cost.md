# v7.0 Phase 5.2 — Audit log viewer + cost reporting

**Status:** ready for plan
**Repo:** `claude-autopilot` (NOT delegance-app)
**Deploys to:** Vercel — confirmed correct per `docs/specs/v7.0-hosted-product-mvp.md` (Stack rationale). claude-autopilot's hosted backend is new infrastructure for a different codebase from delegance-app's ECS — recurring cross-repo confusion that the prior phases also flagged.
**Builds on:** Phase 1 (`audit.events` schema + `audit.append()` chain), Phase 3 (`count_runs_this_month` + `sum_retained_bytes` RPCs), Phase 5.1 (admin layout, role-gating)
**Gates:** Phase 5.3 (WorkOS SSO), Phase 6 (launch prep)

## Goal

Org admins can (a) view their org's tamper-evident audit log with filters and pagination, and (b) export monthly cost reports as CSV broken down by user. Two new pages under `/dashboard/admin/`. Closes the audit half of the original Phase 5 scope; SSO is split into 5.3.

**Codex pass 1 WARNING settled:** "group by run-status" mentioned in v7.0-mvp top-level spec is deferred — Phase 5.2 ships group-by-user only. Adding `run_status` breakdown would change the SQL aggregation shape mid-implementation and isn't load-bearing for the FE-pitch-ready CSV.

## Non-goals

- **PDF export** of cost reports — CSV only. PDF deferred.
- **Hash-chain re-verification UI** ("verify integrity" button). The chain is verified at write time by `audit.append()`; a verifier UI is a 5.4 hardening item.
- **Group-by repo / phase / model / run-status** — the v7.0 mvp spec mentioned these but Phase 5.2 ships **group-by-user only** since `runs` doesn't track per-phase cost or model and run-status breakdown would inflate scope. Codex pass 1 WARNING explicitly settled here.
- **Real-time audit stream / webhooks** — out of scope.
- **Audit retention/expiry policy** — separate compliance discussion. Phase 5.2 reads what's there.
- **Org-tier eligibility gate** — Free orgs can use audit/cost too (admin/owner role gates access). Same as 5.1.
- **Cost report scheduling** (monthly CSV emailed to billing contact) — Phase 6.
- **Org switcher** — Phase 5.3 will add an explicit selector. 5.2 inherits 5.1's "first admin/owner membership" pattern.

## Surfaces

### Backend (3 routes, all under `/api/dashboard/orgs/:orgId/...`)

| Method | Path | Auth | Role | Returns |
|---|---|---|---|---|
| GET | `/audit?cursor=&limit=&action=&actorId=&since=&until=` | session cookie | admin OR owner of orgId | `{ events: [...], nextCursor: string \| null }` |
| GET | `/cost?since=YYYY-MM&until=YYYY-MM&groupBy=user` | session cookie | admin OR owner | `{ rows: [{ key, key_label, run_count, cost_usd_sum, total_bytes_sum }], total: {...}, period: { since, until } }` |
| GET | `/cost.csv?since=&until=&groupBy=user` | session cookie | admin OR owner | `text/csv` body, `Content-Disposition: attachment; filename="cost-<orgId>-<since>-<until>.csv"` (no org name interpolation — codex pass 1+2 CRITICAL) |

Cost endpoints use the same SECURITY DEFINER read RPC pattern as Phase 5.1's `list_org_members_with_emails` — keep authorization in Postgres.

### Frontend (2 pages + 4 components)

- `/dashboard/admin/audit/page.tsx` — Server Component, role-gated. Lists 50 events per page (cursor-based, occurred_at DESC). Filters: action (multi-select), actorId (dropdown of org members), since/until (date pickers).
- `/dashboard/admin/cost/page.tsx` — Server Component. Default period = current month. Shows total cost + run count + storage bytes; per-user table; "Download CSV" button.
- `<AuditTable>` (client) — paginated table with filter chips. Renders `metadata` JSON inline (collapsed). Single-action dropdown (codex pass 1 WARNING — multi-select deferred).
- `<AuditFilterBar>` (client) — controlled filter state, syncs to URL search params.
- `<CostPeriodPicker>` (client) — month-pair selector.
- `<CostTable>` (client) — per-user breakdown + sticky total row.

Sidebar (admin layout from 5.1) gets two more links: **Audit** and **Cost report**.

## Authorization model

**Same as 5.1:** admin/owner of orgId required. Non-admin → 404.

CSRF: not applicable — all endpoints are GET. The `:orgId` path param is validated against active membership before the read RPC fires; non-admin returns 404 (not 403) to avoid enumeration. `assertSameOrigin` is GET-safe to skip per the Phase 5.1 helper docs.

## Data flow

### Audit list

1. GET `/api/dashboard/orgs/:orgId/audit?...`
2. Route validates `orgId` UUID, resolves caller, calls `list_audit_events` RPC
3. RPC: lock-free read, but caller-role check FIRST (raise `not_admin` if not admin/owner)
4. Apply filters; paginate via keyset on `(occurred_at, id)` DESC.

   **Codex pass 1 WARNING — exact predicate:** `ORDER BY occurred_at DESC, id DESC` with the page-2+ filter `(occurred_at, id) < (cursor_occurred_at, cursor_id)`. Also requires index `(organization_id, occurred_at DESC, id DESC)` matching the access pattern. Backdated inserts (occurred_at older than the cursor) can appear in later pages — acceptable for audit-log replay; documented but not "fixed" since `audit.append()` writes occurred_at = NOW() in same transaction so backdating is impossible in practice.

5. Return `{ events: [{ id, action, actorUserId, actorEmail, subjectType, subjectId, metadata, occurredAt, prevHash, thisHash }], nextCursor }` — cursor is base64-encoded `{occurredAt, id}` pair, opaque to client. Tampered cursor → RPC validates cursor parses as `(timestamptz, bigint)`; raise `bad_cursor` (422) on parse failure.

**Filter contract — single-action only.** Codex pass 1 WARNING flagged "multi-select" mismatch; spec settles on single-action filter (`?action=org.member.invited`) for 5.2 simplicity. UI uses single-select dropdown. If users need multi-action, a follow-up adds `action=ANY(p_actions text[])`.

### Cost report (JSON)

1. GET `/api/dashboard/orgs/:orgId/cost?since=2026-04&until=2026-04&groupBy=user`
2. Validate orgId UUID + month strings (`/^\d{4}-(0[1-9]|1[0-2])$/`); 422 `bad_period` on malformed.
3. **Codex pass 1 CRITICAL — period semantics:** both `since` and `until` are **inclusive calendar months in UTC**. Route converts to timestamptz: `sinceTs = since-01T00:00:00Z`, `untilTs = (until + 1 month)-01T00:00:00Z` (exclusive). Validate `sinceTs < untilTs` (which means `since <= until`); 422 `bad_period` if not. Test 14 covers same-month, multi-month, December→January wrap, leap-year February.
4. Call `org_cost_report(p_caller_user_id, p_org_id, p_since, p_until, p_group_by)` RPC: caller-role check, aggregate `runs` for the period grouped by user_id.
5. Return JSON with rows + totals.

### Cost CSV

Same as JSON path, but the route streams CSV with explicit headers. The `org_cost_report` RPC returns the same structured rows; the route formats CSV in JS (no DB-side CSV).

CSV columns: `user_id, email, run_count, cost_usd_sum, duration_ms_sum, total_bytes_sum`.

CSV format (RFC 4180):
- CRLF row terminator
- Cells containing `,` `"` `\n` `\r` wrapped in `"..."` with internal `"` doubled
- UTF-8 with no BOM

**Codex pass 1 CRITICAL — `Content-Disposition` filename injection:** do NOT interpolate raw org name into header. Format: `cost-${orgId}-${since}-${until}.csv` where `since/until` are the validated `YYYY-MM` strings and `orgId` is a validated UUID. Test asserts no double-quote / CRLF / semicolon in the filename header value. Spec test 19b covers.

### Hash-chain integrity

Out of scope for the **viewer** in 5.2. The audit table column `this_hash` is exposed in the JSON for debugging but the UI does not re-verify the chain. Spec test 12 asserts `prev_hash`/`this_hash` are passed through (no UI mangling).

## Migration delta — `data/deltas/20260508160000_phase5_2_audit_cost_rpcs.sql`

Two SECURITY DEFINER RPCs:

```
CREATE FUNCTION public.list_audit_events(
  p_caller_user_id uuid,
  p_org_id uuid,
  p_cursor_occurred_at timestamptz,  -- nullable
  p_cursor_id bigint,                -- nullable
  p_limit int,                       -- 1..200
  p_action text,                     -- nullable
  p_actor_user_id uuid,              -- nullable
  p_since timestamptz,               -- nullable
  p_until timestamptz                -- nullable
) RETURNS jsonb

CREATE FUNCTION public.org_cost_report(
  p_caller_user_id uuid,
  p_org_id uuid,
  p_since timestamptz,
  p_until timestamptz,
  p_group_by text                    -- 'user' (only value supported in 5.2)
) RETURNS jsonb
```

Both:
- `LANGUAGE plpgsql SECURITY DEFINER SET search_path = public, pg_temp`
- `REVOKE ALL ON FUNCTION ... FROM PUBLIC, anon, authenticated; GRANT EXECUTE ... TO service_role;` (codex 5.1 pass 2 CRITICAL pattern)
- Caller role check: `SELECT role FROM memberships WHERE organization_id = p_org_id AND user_id = p_caller_user_id AND status = 'active'`; raise `not_admin` (P0001) if NULL or not in {admin, owner}
- Read-only — no FOR UPDATE locks needed (no last-owner-style aggregate guards)

`list_audit_events` joins `audit.events` to `auth.users` (for actor email) inside the function — same pattern as 5.1's `list_org_members_with_emails`. Avoids exposing `auth` schema to PostgREST.

`org_cost_report` aggregates from `runs`. **Codex pass 1 CRITICAL — `runs.occurred_at` does not exist; use `created_at` for both filter and last-run timestamp. NULL-safe coalesce in aggregates.**

```sql
SELECT
  user_id,
  count(*) AS run_count,
  coalesce(sum(coalesce(cost_usd, 0)), 0) AS cost_usd_sum,
  coalesce(sum(coalesce(duration_ms, 0)), 0) AS duration_ms_sum,
  coalesce(sum(coalesce(total_bytes, 0)), 0) AS total_bytes_sum,
  max(created_at) AS last_run_at
FROM runs
WHERE organization_id = p_org_id
  AND deleted_at IS NULL
  AND created_at >= p_since
  AND created_at <  p_until
GROUP BY user_id
```

LEFT JOIN to `auth.users` for email labels (so deleted users still produce a row with `email = null` per Test 17). `cost_usd` / `duration_ms` / `run_status` come from Phase 4's runs metadata (added 2026-05-08 in `20260508120000_phase4_runs_metadata.sql`).

## Codex passes folded

### Pass 1 — design (3 CRITICAL + 7 WARNING + 2 NOTE — all folded)

1. **CRITICAL: Cost period semantics.** Both `since/until` are inclusive UTC months; route converts to (start-of-since-month, start-of-(until+1)-month) exclusive. Spelled out in data-flow + Test 14.
2. **CRITICAL: `runs.occurred_at` doesn't exist.** SQL uses `created_at` consistently for filter and `last_run_at`; coalesce in SUM aggregates.
3. **CRITICAL: Filename injection.** `Content-Disposition` filename is `cost-<orgId>-<since>-<until>.csv` only. No org name interpolation. Test 19b asserts no `"`/`\r`/`\n`/`;` in header.
4. **WARNING: Keyset predicate spelled out.** `(occurred_at, id) < (cursor)` DESC; index `(organization_id, occurred_at DESC, id DESC)`.
5. **WARNING: action filter single-select** in 5.2 (not multi). Frontend section updated.
6. **WARNING: run-status breakdown deferred.** Group-by-user only; goal text updated.
7. **WARNING: deleted users.** LEFT JOIN auth.users; `email` may be null. Test 17.
8. **WARNING: CSV format.** RFC 4180 with CRLF, UTF-8 no BOM.
9. **WARNING: limit clamping** server-side (Test 9).
10. **NOTE: spreadsheet formula injection.** Cells starting with `=`/`+`/`-`/`@` aren't sanitized — documented as accepted risk for raw-data export. Phase 5.4 may add hardened mode.

### Pass 2 — final (2 CRITICAL + 7 WARNING + 2 NOTE — all folded)

1. **CRITICAL: Vercel deployment correctness.** Confirmed accurate per top-level v7.0 spec; clarified inline so future readers don't second-guess.
2. **CRITICAL: filename header contradiction.** Surfaces table updated to match security section: `cost-<orgId>-<since>-<until>.csv`.
3. **WARNING: cursor validation responsibility** is the route's, not the RPC's. Route base64-decodes + JSON-parses + validates `(timestamptz, bigint)` shape → 422 `bad_cursor` before RPC. RPC just accepts typed nullable params.
4. **WARNING: SQLSTATE error contract.** All RPC raises use `P0001` with distinct keyword messages (`not_admin`, `bad_group_by`); routes parse `error.message` to map. `bad_cursor`/`bad_period`/`malformed_params` are route-side validation failures, never reach RPC.
5. **WARNING: cost JSON shape unified.** Per-row: `{ user_id, email, run_count, cost_usd_sum, duration_ms_sum, total_bytes_sum, last_run_at }`. Total: `{ run_count, cost_usd_sum, duration_ms_sum, total_bytes_sum }`. CSV columns omit `last_run_at` for compactness; documented.
6. **WARNING: audit since/until parsing.** ISO 8601 UTC timestamps only (`Z` suffix required). `YYYY-MM-DD` rejected as 422 to avoid timezone ambiguity. Test 7 + new Test 7b for malformed.
7. **WARNING: cache headers.** All 3 routes return `Cache-Control: private, no-store`. Pages declare `export const dynamic = 'force-dynamic'`.
8. **NOTE: privilege test executable.** Migration grep test (#27) + plan-time follow-up to add runtime privilege test in 5.4 hardening (deferred — would require ephemeral DB, same pattern as Phase 1's `db-tests.yml`).
9. **NOTE: negative-contract tests added.** `bad_cursor`, `limit=-1`, `groupBy=repo`, missing period (defaults to current month UTC) covered in Tests 7b, 9, 14, plus a new Test 12b for default-period behavior.
